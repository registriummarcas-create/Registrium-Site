# Contribuição

## Fluxo

1. Crie uma issue.
2. Abra uma branch `feature/*`, `fix/*`, `content/*`, `legal/*`, `chore/*` ou `hotfix/*`.
3. Faça commits pequenos no padrão Conventional Commits.
4. Execute `pnpm check`.
5. Abra pull request para `main`.
6. Use squash merge após revisão e CI verde.

## Commits

Exemplos:

```text
feat(site): implementa seção principal
feat(worker): cria endpoint do Instagram
fix(site): corrige CTA no Safari
content(legal): atualiza política de privacidade
```
